<threat_model>
## Security Threat Model

### T-01-01: Path confusion or accidental writes outside intended directory
- Severity: low
- Mitigation: Treat `--session-dir` as an explicit user-supplied filesystem path and pass it directly to `JsonlSessionStore`, which creates the selected directory. Do not add implicit shell execution or path interpolation.
- Verification: CLI tests use `tmp_path` and assert logs are written only under the expected directory.

### T-01-02: Sensitive data in persisted events
- Severity: medium
- Mitigation: This phase persists the same structured `SessionEvent` payloads the runtime already emits. README must describe the logs as audit artifacts and remind users that `.citrus/` is gitignored because local config/logs may be sensitive.
- Verification: README contains session persistence documentation and `.gitignore` already ignores `.citrus/`.
</threat_model>
